Removing Duplicates Objects from Array

I am firm believer of the fact that programing languages are tools and to be good at solving problems, knowing you tool will make your life so much easier. Which holds true in the case of this problem as well.

So, today I came across this article https://bitsofco.de/removing-duplicate-objects-from-an-array-is-hard/ which talks about how it is difficult (as of computationally) to remove duplicate elements from an array in case of non-primitive types such as objects.

Now the solution proposed by the author is not a great one, I agree but he shows of a great trick I just learned for primitive data and I am so gone use this from now. But the point is that this indeed is a problem to thing about. You can get this idea if you google for this problem, you will find so much content with all kinds of approaches and all complexity solutions. So, its a well know problem people encounter on day to day basic.

Now how can we solve this problem in a way which takes advantage of your tool.

This is my implementation to solve the same problem using JavaScript Symbols(https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Symbol). Basically Symbols are a way to generate values which are internally represented as different values upon each function call. But Symbol.for(‘cat’) allows you to make a global symbol and upon each construction will give you the same value.

Now, using that approach, we first convert the array into an array of symbols and then we can use the same idea of having primitives and removing duplicates and that’s it basically.

Afterwards I did some tests just for fun and turns out this solution is pretty scale able too. Here are some results:-

  • For 100 elements or so, it takes about 5.5ms where as the approach used in the original article takes 2.2ms.
  • For 500 elements or so, it takes 5.7ms whereas the other algorithm takes 11.7ms.
  • For 1500 elements or so, it takes 3.886ms whereas the other algorithm takes 32.82ms.
  • For 8000 elements or so, it takes 5.57ms whereas the other algorithm takes 60.71ms.
